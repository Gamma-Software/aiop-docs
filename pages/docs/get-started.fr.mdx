# Démarrage

Cette introduction rapide vous aide à commencer à utiliser Aiop rapidement. N’hésitez pas à consulter la [vidéo](/docs/demo) démonstration qui reprend les éléments de cette page.

Vous pouvez installer Aiop dans un environnement local Python ou avec Docker. Pour une installation locale, suivez les instructions ci-dessous. Pour une installation avec Docker, consultez cette [section](/docs/get-started#docker).

## Préréquis

### Python

<Steps>

### Installation de Python

Installez la version recommandée de Python 3.11.8 en utilisant votre gestionnaire de paquets. Mettez à jour votre gestionnaire de paquets et installez les dépendances nécessaires. Puis ajoutez le dépôt de deadsnakes pour installer les versions de Python.

```bash copy
sudo apt-get update -y
sudo apt-get upgrade -y
sudo apt-get install build-essential libssl-dev libffi-dev python3-dev
sudo add-apt-repository ppa:deadsnakes/ppa
```

Ensuite installez la version de python de votre choix. Veuillez installer les versions supérieurs à 3.11. Nous recommandons d’installer la version 3.11.8.

```bash copy
sudo apt install python3.11
```

<Callout type="info" emoji="ℹ️">
  Vous pouvez toutefois installer la dernière version 3.12. En revanche, nous ne garantissons pas que Aiop fonctione.
</Callout>

Vérifiez la version de python installée avec la commande `python3 --version`.

### Installation des dépendances

Additionnellement, installez des outils de développement et des bibliothèques pour Python que Aiop utilisera.

```bash copy
sudo apt-get install curl iputils-ping -y
```

### Installation de Pip
Si vous n’avez pas pip d’installez, vous pouvez l’installer avec la commande suivante:

```bash copy
sudo apt-get install python3-pip -y
```
</Steps>

### Alias

Pour simplifier l’utilisation de python, il est recommandé de créer des raccourcis des commandes `python` et `pip` à `python3` et `pip3` respectivement:

```bash copy
ln -s /usr/bin/python3 /usr/bin/python
ln -s /usr/bin/pip3 /usr/bin/pip
```

## Installation

<Steps>

### Créer un compte aiop (Optionnel)

Commencez par créer un compte aiop. Si vous n’avez pas de compte, appuyiez sur le bouton ci-dessous pour en créer un.



### Installer aiop
Installer aiop est simple. Ouvrez un terminal et exécutez la commande suivante:

```bash copy
pip install aiop --index-url https://gitlab.com/api/v4/projects/52727704/packages/pypi/simple
```

Cela installera aiop et toutes ses dépendances.


### Télécharger un modèle de playbook

Pour utiliser aiop, vous avez besoin d’un [playbook](/docs/vocabulary#playbook). Vous pouvez installer un modèle démonstratif de playbook en exécutant la commande suivante:

```bash copy
aiop demo ./my-demo-playbook
```

Cela téléchargera le modèle de playbook dans le dossier `my-playbook`.

### Générer son premier package

Vous y êtes enfin ! Vous pouvez maintenant générer votre premier package en exécutant la commande suivante:

```bash copy
cd my-playbook
aiop build archi1/model_range1/model1 --apps app1 --envs dev --contexts context1
```

<Callout type="info" emoji="✅">
  Terminé, vous venez de générer votre premier package !
</Callout>

Vous pouvez maintenant voir le package généré dans le dossier `.build` à la racine du playbook.

</Steps>

## Docker

Si vous préférez utiliser Docker, vous pouvez l’installer en suivant les instructions ci-dessous.

<Steps>

### Installer Docker

Installer Docker sur votre machine:

- [Docker](https://docs.docker.com/get-docker/)

### Télécharger l’image docker

Téléchargez l’image docker aiop en exécutant la commande suivante:

```bash copy
docker pull registry.gitlab.com/aiop/aiop:latest
```

Pour une version spécifique, remplacez `latest` par la version souhaitée.

```bash copy
docker pull registry.gitlab.com/aiop/aiop:0.3.1
```

Pour une version en cours de développement:

```bash copy
docker pull registry.gitlab.com/aiop/aiop/dev:latest
```

### (Recommandé / optionnel) changer le nom de l’image

Pour simplifier l’utilisation de l’image docker, vous pouvez renommer l’image en exécutant la commande suivante:

```bash copy
docker tag registry.gitlab.com/aiop/aiop:latest aiop
```

### Exécuter aiop

Pour exécuter aiop, exécutez la commande suivante:

```bash copy
docker run -it registry.gitlab.com/aiop/aiop:latest
```

<Callout type="info" emoji="ℹ️">
  L'option `-it` est utilisée pour exécuter le conteneur en mode interactif et allouer un pseudo-TTY.
</Callout>

Ou, si vous avez renommé l’image:

```bash copy
docker run -it aiop
```

### Générer son premier package

Vous pouvez maintenant générer votre premier package en exécutant la commande suivante:

```bash copy
docker run -it -v {local_playbook}:/playbook -v {path_to_aiop.yml}:/config/aiop.yml aiop build {build_args}
```

Vous observerez 3 arguments:

- `-v {local_playbook}:/playbook`: `local_playbook` est le chemin vers votre playbook sur votre machine locale.
- `-v {path_to_aiop.yml}:/config/aiop.yml`: `path_to_aiop.yml` est le chemin vers votre fichier aiop.yml sur votre machine locale.
- `aiop build {build_args}`: `build_args` sont les arguments de génération de package.

</Steps>
