# Vocabulaire

AIOP utilise un vocabulaire spécifique pour décrire ces concepts.

## AIOP

`All-In-One-Packager` génère des packages uniformes pour différentes plateformes à l'aide d'une simple déclaration de source. "Packages as Code".

## Package

Un package logiciel est un ensemble de fichiers et d’instructions pour installer, configurer, utiliser et gérer un logiciel ou une bibliothèque sur un ordinateur.

## Leaptech

Leaptech est le nom de l'entreprise qui développe AIOP.

## AIOP core

C’est le coeur de AIOP. Il contient les fonctionnalités de base pour la gestion des ressources. Il est responsable de la sélection des ressources en fonction de la cible de l’utilisateur.

## AIOP cli

AIOP est accessible via une interface en ligne de commande (CLI-Command Line Interface en anglais). [En savoir plus](/docs/cli-index)

## AIOP lib

Un [playbook](#playbook) est extensible via des scripts python personnalisés. Ces scripts doivent inclure des dépendances à la librairie de AIOP.

## AIOP plugin

AIOP est extensible via des plugins. Un plugin est une extension qui permet d’ajouter des fonctionnalités à AIOP. [En savoir plus](/docs/aiop-lib/plugins)

## Registres d'artefacts

Les registres d'artefacts sont des systèmes de stockage de fichiers binaires. Ils sont utilisés pour stocker les [artefacts](#artefact) générés par AIOP. [En savoir plus](/docs/declarations/cookbook/remote)

## Artefact

Les artefacts sont des éléments qui représentent les entités physiques dans un système logiciel. Les artefacts représentent des unités physiques d'implémentation, telles que des fichiers exécutables, des bibliothèques, des composants de logiciel (binaires ou fichiers de configuration), des documents, et bases de documents.

## Cible (Système)

Une cible est une combinaison de paramètres pour lequels vous souhaitez générer un package. C’est le [playbook](#playbook) qui décrit la combinaison de paramètres pour une cible donnée. [En savoir plus](/docs/configurations/inventory)

## Linting

Le linting est un processus qui consiste à analyser les déclarations pour trouver des erreurs, des bogues, des erreurs stylistiques et des constructions suspectes. Nous distinguons deux processus différents:

- **Linting Static:** Le plus classique, il permet de vérifier la syntaxe des fichiers de déclaration.
- **Linting Dynamic:** Il permet de vérifier la cohérence des déclarations en fonction des ressources disponibles sur le système.

[En savoir plus](/docs/cli/lint)

## AML

> AIOP Markup Language

C'est le fichier de déclaration de ressources. Ce fichier se base sur le langage de balisage YAML. [En savoir plus](/docs/declarations/overview)

## Déclaration

Une déclaration correspond à une ressources décrite dans un [.aml](#aml). [En savoir plus](/docs/declarations/overview)

## Inventaire

Un inventaire est une liste de cibles. Pour chaque cible, des packages peuvent être généré. [En savoir plus](/docs/configurations/inventory)

## Package As Code (PAC)

Le package as code est une méthode de gestion de packages qui consiste à décrire les packages à l'aide d’un fichier textuel compréhensible par tous.

## Playbook

Le coeur de votre projet qui contient toutes les ressources (textes, configurations, binaires...) ainsi que les fichiers déclaratifs de AIOP appelées [`.aml`](#aml). [En savoir plus](/docs/configurations/playbook)

## Markdown

Markdown est un langage de balisage léger créé en 2004 par John Gruber avec l'aide d'Aaron Swartz. Son but est d'offrir une syntaxe facile à lire et à écrire.

## Git

Git est un système de contrôle de version qui permet aux développeurs de suivre les modifications apportées au code, de travailler en équipe sur un même projet, et de fusionner facilement les contributions. Ses avantages incluent une gestion efficace des versions, des branches pour isoler les fonctionnalités, une performance rapide même sur de grands dépôts, une distribution décentralisée pour un travail hors ligne, une flexibilité dans les workflows, et un écosystème robuste avec des services comme GitHub et GitLab. En résumé, Git améliore la collaboration, la gestion et le suivi des projets de développement logiciel. [En savoir plus](https://git-scm.com)

## FMPP

FMPP (FreeMarker-based file PreProcessor) est un outil de génération de fichiers basé sur des modèles. Il permet de générer des fichiers en remplaçant des balises de modèle par des valeurs définies dans un fichier de configuration. [En savoir plus](https://fmpp.sourceforge.net/license.html)

## Jinja

Jinja est un moteur de création de modèles rapide, expressif et extensible. Des espaces réservés spéciaux dans le modèle permettent d'écrire un code similaire à la syntaxe Python. Le modèle reçoit ensuite les données nécessaires au rendu du document final. Pour en savoir plus sur ses fonction pré-définies, voici [la documentation officielle](https://jinja.palletsprojects.com/en/3.1.x/templates/#filters)
