# Configuration utilisateur

Aiop permet à l’utilisateur de personnaliser le comportement de l’application en modifiant le fichier de configuration. Le fichier de configuration est situé dans le répertoire de l’utilisateur:
- Unix: `~/.aiop/config.yaml`
- Windows: `%HOME%\.aiop\config.yaml`

<Callout type="info" emoji="ℹ️">
  À l’installation de Aiop, un fichier de configuration par défaut est créé. Il est possible de le [modifier](#personalisation) pour personnaliser le comportement de l’application.
</Callout>

Le fichier de configurartion est au format YAML. Il est possible de le modifier avec un éditeur de texte ou un éditeur de code. Il sera ensuite lu et vérifié par Aiop pour personnaliser son comportement.

## Structure du fichier de configuration

Voici la structure du fichier de configuration. Nous allons ensuite détailler chaque section.

```yaml
build_timeout: {valeur numérique}
check_update: {true|false}
default_package_path: {chemin absolu ou relatif à la racine du playbook}
download_retries: {valeur numérique}
plugin_config: {dictionnaire de configuration spécifique à un plugin}
logs:
  level: {CRITICAL|FATAL|ERROR|WARNING|WARN|INFO|DEBUG|DISABLED}
  output: {stdout|file}
  pathdir: {chemin absolu}
repositories:
  - instance: {nom de l'instance}
    username: {nom d'utilisateur}
    password: {mot de passe}
    auth_type: {basic|api_key}
license:
  key: {clé de licence}
  username: {nom d'utilisateur}
  password: {mot de passe}
  api_token: {token d'API}
```

**build_timeout** (int): Durée maximale en secondes pour la construction d’un package.

**check_update** (bool): Activer ou désactiver la vérification des mises à jour à l’exécution de l’application.

**default_package_path** (str): Chemin par défaut ou stocker les packages générés. Il peut être absolu ou relatif à la racine du playbook.

**download_retries** (int): Nombre de tentatives pour télécharger une resource distante.

**plugin_config** (dict): Configurations spécifique aux plugins. Se fier à la documentation du plugin pour plus d’informations. [Contacter le support](/support) pour plus d’informations.

**logs** (dict): Configuration des logs.
    - **level** (str): Niveau de log. Peut être CRITICAL, FATAL, ERROR, WARNING, WARN, INFO, DEBUG ou DISABLED.
    - **output** (str): Sortie des logs. Peut être `stdout` pour l’afficher dans la console ou `file` dans un fichier.
    - **pathdir** (str): Chemin absolu du répertoire de stockage des logs.

**repositories** (list): Liste des dépôts distants.
    - **instance** (str): Nom de l’instance. Utilisée pour identifier le dépôt.
    - **username** (str): Nom d’utilisateur pour l’authentification au dépôt.
    - **password** (str): Mot de passe pour l’authentification au dépôt.
    - **auth_type** (str): Type d’authentification. Peut être `basic` ou `api_key`. Dans le cas de `api_key`, le `password` est la clé d’API et l’`utilisateur` est ignoré.

**license** (dict): Informations de licence.
    - **key** (str): Clé de licence.
    - **username** (str): Nom d’utilisateur pour l’authentification à Aiop.
    - **password** (str): Mot de passe pour l’authentification à Aiop.
    - **api_token** (str): Token d’API pour l’authentification à Aiop. Si `api_token` est défini, `username` et `password` sont ignorés.

## Personalisation

Pour personnaliser le comportement de l’application, il suffit d’ouvrir et de modifier le fichier de configuration.

```yaml
build_timeout: {valeur numérique}
check_update: {true|false}
default_package_path: {chemin absolu}
download_retries: {valeur numérique}
plugin_config:
  ingenico_sdk_path: {chemin absolu}
logs:
  level: {CRITICAL|FATAL|ERROR|WARNING|WARN|INFO|DEBUG|DISABLED}
  output: {stdout|stderr|file}
  pathdir: {chemin absolu}
repositories:
  - instance: {nom de l'instance}
    username: {nom d'utilisateur}
    password: {mot de passe}
    auth_type: {basic|api_key}
license:
  key: {clé de licence}
  username: {nom d'utilisateur}
  password: {mot de passe}
  api_token: {token d'API}
```
