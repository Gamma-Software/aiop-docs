# Commande de Linting

La commande de linting permet de lancer des scripts de vérification du playbook, de ses paramètres, de ses déclarations et de ses resources. Eventuellement, il est possible de vérifier le système cible et les ressources compatibles. De plus il est possible d'executer des scripts de vérifications personnalisées inclues dans le playbook.

## Commande `lint`

Pour lancer la commande de `lint`, il faut que l'utilisateur se place dans le répertoire du playbook.
Cette commande permet de vérifier le playbook, ses paramètres, et la syntaxe de ses déclarations et l'existance de ses resources.

```bash
aiop lint
```

En lançant la commande d'aide voici les options disponibles:

```ansi
usage: aiop lint [-h]

options:
  -h, --help  show this help message and exit
```

## Commande `lint_target`

Pour lancer la commande de `lint_target`, il faut que l'utilisateur se place dans le répertoire du playbook.
La commande `lint_target`permet de vérifier le playbook, ses paramètres, et la syntaxe de ses déclarations et l'existance de ses resources et de leur compatibilités avec le système cible.

```bash
aiop lint_target {system_target} {params}
```
> `system_target` : La cible système pour laquelle valider le playbook

> `params` : Les paramètres de compatibilité de la cible système. Ce paramètre obligatoire sera différent selon le playbook et ses configurations car il est défini par le fichier de configuration `.aiop/inventory_config.yml` dans le champ `compatibility` ([En savoir plus](/docs/configurations/inventory)).

En lançant la commande d'aide voici les options disponibles:

```ansi
usage: aiop lint_target [-h] [--mode MODE] system_target [apps ...] context

positional arguments:
  system_target  System target for which to validate the playbook
  params         Compatibility parameters

options:
  -h, --help     show this help message and exit
  --mode MODE    Mode of the created output (default: dev)
```
