# Commande de Linting

La commande de linting permet de lancer des scripts de vérification du playbook, de ses paramètres, de ses déclarations et de ses ressources. Éventuellement, il est possible de vérifier le système cible et les ressources compatibles. De plus il est possible d'executer des scripts de vérifications personnalisées inclues dans le playbook.

## Commande `lint`

Pour lancer la commande de `lint`, il faut que l'utilisateur se place dans le répertoire du playbook.
Cette commande permet de vérifier le playbook, ses paramètres, et la syntaxe de ses déclarations et l'existence de ses ressources.

```bash
aiop lint
```

En lançant la commande d'aide voici les options disponibles:

```ansi
usage: aiop lint [-h] [-v] [-sll STDOUT_LOG_LEVEL] [-fll FILE_LOG_LEVEL] [-hp]

options:
  -h, --help            show this help message and exit
  -v, --verbose         Force the verbosity of the logs to DEBUG. Thus overriding the verbosity set in the configuration file.
  -sll STDOUT_LOG_LEVEL, --stdout_log_level STDOUT_LOG_LEVEL
                        Sets and overrides the stdoutlog level of the logs.
  -fll FILE_LOG_LEVEL, --file_log_level FILE_LOG_LEVEL
                        Sets and overrides the file_log level of the logs.
  -hp, --hide_progress  Hide the progress bar.
```

## Commande `lint_target`

Pour lancer la commande de `lint_target`, il faut que l'utilisateur se place dans le répertoire du playbook.
La commande `lint_target`permet de vérifier le playbook, ses paramètres, et la syntaxe de ses déclarations et l'existence de ses ressources et de leur compatibilités avec le système cible.

```bash
aiop lint_target {system_target} {params}
```
> `system_target` : Le système cible pour laquelle valider le playbook

> `params` : Les paramètres de compatibilité de la cible système. Ce paramètre obligatoire sera différent selon le playbook et ses configurations car il est défini par le fichier de configuration `.aiop/inventory_config.yml` dans le champ `compatibility` ([En savoir plus](/docs/configurations/inventory)).

En lançant la commande d'aide voici les options disponibles:

```ansi
usage: aiop lint_target [-h] --compat_args ARGS [ARGS ...] [--no_verify] [--retries RETRIES] [--no_caching] [-v] [-sll STDOUT_LOG_LEVEL]
                        [-fll FILE_LOG_LEVEL] [-hp]
                        system_target

positional arguments:
  system_target         System target for which to create a package

options:
  -h, --help            show this help message and exit
  --compat_args ARGS [ARGS ...]
                      List of compatibility arguments for the target system
  --no_verify           Do not verify the Artifact integrity. It will not betaken into account if `--no_caching` is not set or the Artifact is not in cache.
                        (default: False, means verifythe artifact integrity)
  --retries RETRIES     Number of retries when downloading an artifact.One retry means that the download was not successful because of an invalid file integrity
                        (checksum).This parameter will override [download_retries] in the user configuration file. - default: 3
  --no_caching          The binaries are copied to the destination folder (not from the cache)
  -v, --verbose         Force the verbosity of the logs to DEBUG. Thus overriding the verbosity set in the configuration file.
  -sll STDOUT_LOG_LEVEL, --stdout_log_level STDOUT_LOG_LEVEL
                        Sets and overrides the stdoutlog level of the logs.
  -fll FILE_LOG_LEVEL, --file_log_level FILE_LOG_LEVEL
                        Sets and overrides the file_log level of the logs.
  -hp, --hide_progress  Hide the progress bar.
```
